--- 1. DATABASE SETUP (MongoDB Atlas) ---
Since your `.env` currently points to `mongodb://localhost:27017/me4_ecommerce`, you will need a cloud database for live deployment.
1. Go to https://www.mongodb.com/cloud/atlas and sign up/login.
2. Build a Database (Choose the "FREE" shared cluster).
3. Once created, click "Connect". 
4. Select "Drivers" (Node.js).
5. Copy the connection string provided (it looks like: mongodb+srv://<username>:<password>@cluster0.xxxxx.mongodb.net/?retryWrites=true&w=majority).
6. Replace `<username>` and `<password>` with the database user you create in Atlas. Also define the database name before `?retryWrites`.
7. Update your `.env` file LOCALLY with this new string to test it, and save it for step #3 (Vercel).

--- 2. ADDING MORE APIs & DATA ---
1. To insert test products, you can use Postman to send POST requests or do it directly via MongoDB Compass using the connection string.
2. The authentication API handles login natively using JWTs.
3. The Predictive Analytics API is fully simulated via `analyticsController.js`. It requires that a User has previous Orders to suggest accurate items, otherwise it falls back to highly-rated products! Send a few test orders to see it adapt!

--- 3. DEPLOYING TO VERCEL (FREE HOSTING) ---
Vercel is perfect for this! It can host your backend APIs and serve your static frontend files (`/public`) easily.

Step A: Create `vercel.json` (Required for Express App on Vercel)
Create a new file called `vercel.json` in the root of your project directory (`ME4 ecommerce-backend 1.0/`) and paste this exact code:

{
  "version": 2,
  "builds": [
    {
      "src": "index.js",
      "use": "@vercel/node"
    }
  ],
  "routes": [
    {
      "src": "/(.*)",
      "dest": "index.js"
    }
  ]
}

Step B: Push Code to GitHub
Ensure all your modified files are committed and pushed to your repository: https://github.com/ashfaaqkt/E-Commerce-Backend-System---Entri---MERN

Step C: Deploying on Vercel Website
1. Go to https://vercel.com and login with your GitHub account.
2. Click "Add New..." -> "Project".
3. Import your "E-Commerce-Backend-System---Entri---MERN" repository.
4. Framework Preset: Leave as "Other".
5. Root Directory: If your code is inside the "ME4 ecommerce-backend 1.0" folder in Git, set the Root Directory to that folder.
6. Environment Variables (CRITICAL):
   Add the following variables exactly as they appear in your local `.env`:
   - MONGODB_URI (Paste the Atlas string from Step 1)
   - JWT_SECRET (e.g. me4_secret_key_super_secure_2026)
   - PORT (Vercel handles ports, but good to add: 5000)
7. Click "Deploy". 

Wait 1-2 minutes for Vercel to build the architecture. Once finished, click on the domain they give you (e.g., something.vercel.app).
Congratulations! Your futuristic Landing UI will render, and submitting forms will safely communicate with the `api/` express routes!
